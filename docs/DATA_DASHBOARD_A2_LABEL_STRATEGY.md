# Data Dashboard - A2 Navigation Label Strategy

Task: Architecture A2 (UX+RED)  
Objective: finalize action-oriented naming language for top-level navigation.

## Final Top-Level Labels

Primary menu labels (English-first dashboard brand):

1. Command Center
2. Profile
3. Intelligence
4. Stack
5. Economics Lens
6. Vision Lab

## Why these labels

- `Command Center`: positions Home as an operational dashboard, not a blog feed.
- `Profile`: communicates executive identity and capabilities (CV + positioning).
- `Intelligence`: reframes posts as decision-oriented insights.
- `Stack`: reframes tags/tools as an applied capability map.
- `Economics Lens`: preserves your core economics authority explicitly.
- `Vision Lab`: marks CV as an emerging track without overstating specialization.

## Microcopy Rules

1. Use outcome language, not archive language.
   - Prefer `Insights`, `Decisions`, `Impact`, `Systems`.
2. Keep labels short (1-2 words), high-signal, no jargon stacking.
3. Avoid ambiguous terms (`Blog`, `Misc`, `Stuff`, `Notes`).
4. Prioritize domain honesty:
   - Analysis / ML / BI / Economics as primary authority.
   - CV clearly framed as growth lane (`Lab`).
5. CTA tone:
   - Action first (`Explore`, `View`, `Compare`, `Read`).
6. Metadata language in cards:
   - `Domain`, `Level`, `Reading time`, `Business impact`.

## Bilingual Fallback Guidance

Baseline:

- UI navigation remains English-first for premium dashboard positioning.

Spanish fallback mappings (for progressive localization):

- Command Center -> Centro de Comando
- Profile -> Perfil
- Intelligence -> Inteligencia
- Stack -> Stack Tecnologico
- Economics Lens -> Lente Economico
- Vision Lab -> Laboratorio de Vision

Fallback policy:

1. Keep one active UI language per session.
2. If Spanish fallback is enabled, apply full-label consistency (no mixed nav).
3. Preserve domain nouns in English only where clarity improves recognition (`Stack`, `Vision` allowed).
